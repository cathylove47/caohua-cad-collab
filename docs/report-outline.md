# 课程设计报告大纲

## 1. 课题背景与研究意义

- 机械 CAD 系统在工程设计中的重要作用
- 云计算与协同设计的发展趋势
- 传统单机 CAD 在多人实时协同方面的局限
- 设计并实现一个可演示的云协同 CAD MVP 的现实意义

## 2. 需求分析

### 2.1 功能需求

- 基础三维建模
- 草图对象表达
- 挤出生成实体
- 多用户协同编辑
- 在线用户与光标显示
- 保存、加载、版本管理

### 2.2 非功能需求

- 本地稳定运行
- 易于演示
- 架构清晰
- 易于扩展

## 3. 总体方案设计

- 技术栈选择理由
- 前端重计算、后端轻逻辑架构
- 房间协同模型
- JSON 持久化与版本快照方案

## 4. 系统架构设计

### 4.1 总体架构图

- 前端
- 后端
- WebSocket
- 本地文件存储

### 4.2 模块划分

- 登录模块
- CAD 编辑模块
- 协同通信模块
- 持久化模块
- 版本管理模块

## 5. 关键功能实现

### 5.1 CAD 界面设计

- Ribbon 工具栏
- Feature Tree
- 3D 视图区
- 属性面板
- 在线状态栏

### 5.2 基础建模实现

- Box / Cylinder / Sphere
- Sketch Line / Circle / Rectangle
- Extrude
- 简化 Cut

### 5.3 协同机制实现

- 用户加入房间
- WebSocket 消息格式
- 操作广播
- 光标同步
- 冲突策略：最后写入优先

### 5.4 存储与版本管理

- Save / Load
- 本地 JSON 文件组织
- 版本快照生成
- 版本恢复

## 6. 系统测试与演示结果

### 6.1 功能测试

- 双窗口进入同一房间
- 新建模型同步
- 参数修改同步
- Save 后刷新再 Load
- Undo / Redo
- 版本恢复

### 6.2 结果分析

- 哪些目标已达到
- 协同延迟与稳定性表现
- MVP 的优缺点

## 7. 项目创新点与工程取舍

- 以较低复杂度实现协同 CAD MVP
- 前端渲染与后端协同的职责分离
- 面向课程设计的稳定性优先策略

## 8. 不足与后续优化

- 未集成工业级几何内核
- Cut 为简化实现
- 冲突控制较简单
- 无正式鉴权

后续方向：

- OpenCASCADE/Wasm
- PostgreSQL
- OBS
- JWT
- 华为云部署

- 总结系统完成情况
- 总结架构设计价值
