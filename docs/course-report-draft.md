# 《基于云的协同机械 CAD 系统设计与实现》课程设计报告初稿

## 摘要

随着云计算、Web 图形渲染以及协同编辑技术的发展，传统单机式机械 CAD 软件在多人实时协同、轻量部署和跨终端访问方面逐渐暴露出局限性。针对课程设计“基于云的协同机械 CAD 系统设计与实现”的要求，本文设计并实现了一个面向本地演示和课程答辩的最小可交付系统（MVP）。

系统以前端 React + TypeScript + Vite 为开发基础，采用 Three.js 实现三维图形渲染，使用 Zustand 维护前端建模状态，后端基于 Node.js + Express + WebSocket 提供房间协同与状态广播服务，并通过本地 JSON 文件实现项目保存与版本快照。系统支持 Box、Cylinder、Sphere 等基础实体建模，支持 Line、Circle、Rectangle 草图表达，并实现了基于 Rectangle/Circle 的 Extrude 操作。同时，系统支持双窗口同房间协同编辑、在线用户显示、光标同步、Undo/Redo、Save/Load、版本恢复等功能。为增强创新性，系统增加了 Smart Assist 自然语言辅助建模入口，并在架构中预留了 OpenCASCADE/Wasm 与 AI 辅助设计的扩展接口。

实验结果表明，该系统能够在本地稳定运行，能够完成课程设计要求中的核心功能演示，并且具有结构清晰、易于讲解和便于后续扩展的特点。

关键词：云 CAD；协同设计；Three.js；WebSocket；版本管理；Wasm 扩展

## 1. 引言

### 1.1 研究背景

机械 CAD（Computer Aided Design）系统广泛应用于零件建模、结构设计、工程仿真和制造准备等场景。传统 CAD 软件通常以单机桌面模式为主，虽然功能强大，但在以下方面存在不足：

- 多用户实时协同能力较弱；
- 跨平台与轻量部署成本较高；
- 课程设计环境下实现复杂工业级内核难度大。

随着浏览器图形能力提升以及云端协同技术成熟，基于 Web 的协同 CAD 系统成为一个具有代表性的研究与工程实践方向。

### 1.2 研究目标

本课设并不追求工业级 CAD 内核的完整实现，而是聚焦以下目标：

- 设计一个可本地稳定运行的云协同 CAD MVP；
- 实现基础建模、实时协同、保存与版本管理闭环；
- 构建结构清晰、便于报告书写和答辩讲解的系统架构；
- 为 OpenCASCADE/Wasm、数据库和 AI 辅助设计预留扩展接口。

## 2. 需求分析

### 2.1 功能需求

根据课题要求，系统需要实现以下功能：

1. 前端 CAD 界面
   - Ribbon 风格工具栏
   - Feature Tree
   - 3D 视图区
   - 属性面板
   - 在线用户与连接状态显示
2. 基础建模
   - Box、Cylinder、Sphere
   - Line、Circle、Rectangle 草图
   - Extrude
   - 简化 Cut
3. 协同功能
   - 房间机制
   - WebSocket 实时同步
   - 在线用户与远端光标
4. 项目存储
   - Save / Load
   - 版本快照与恢复
   - Undo / Redo

### 2.2 非功能需求

- 本地运行稳定
- 不依赖 Docker
- 不要求远程部署
- 架构清晰、便于答辩
- 可扩展到工业级几何内核和云端存储

## 3. 系统总体设计

### 3.1 技术栈选型

本系统的技术选型如下：

- 前端：React + TypeScript + Vite
- 三维渲染：Three.js
- 状态管理：Zustand
- 后端：Node.js + Express + ws
- 存储：本地 JSON 文件

选择理由如下：

- React + TypeScript 组件化开发效率高，适合构建结构清晰的前端系统；
- Three.js 可快速完成三维几何体渲染；
- WebSocket 适合实现多人实时操作广播；
- JSON 存储简单直接，适合课程设计本地环境；
- 以上技术栈本地部署门槛低，便于演示。

### 3.2 架构思想

本系统采用“前端重计算、后端轻协同”的架构：

- 前端负责建模逻辑、交互逻辑、视图渲染和局部操作历史；
- 后端负责房间状态维护、协同消息广播、保存与版本化；
- 存储层负责将当前项目状态写入 JSON 文件，并生成版本快照。

这种设计降低了后端复杂度，同时便于未来替换前端几何内核。

### 4.1 总体架构

系统总体架构由三部分组成：

### 4.2 模块划分

前端模块：

- `StatusPanel`

后端模块：

- `server.ts`

- `storage.ts`

### 4.3 数据流设计

系统的典型数据流如下：

1. 用户输入用户名与房间号；
2. 前端连接 WebSocket 并加入房间；
3. 用户执行建模或编辑操作；
4. 前端本地更新状态并发送操作消息；
5. 后端更新房间状态并广播给其他客户端；
6. 用户点击 Save 后，后端将状态持久化到 JSON 文件并生成版本快照。

## 5. 关键功能实现

### 5.1 CAD 界面实现

系统前端采用典型三栏 CAD 布局：

- 顶部为 Ribbon 风格工具栏；
- 左侧为 Feature Tree；
- 中间为 Three.js 三维视图区；
- 右侧为属性面板；
- 底部为房间状态与在线用户信息。

这样的布局有利于符合 CAD 系统用户认知，也便于答辩时分模块讲解。

### 5.2 基础建模实现

#### 5.2.1 Primitive 实体

系统支持以下基础实体：

- Box
- Cylinder
- Sphere

其本质是利用 Three.js 中的标准几何体类：

- `BoxGeometry`
- `CylinderGeometry`
- `SphereGeometry`

#### 5.2.2 草图对象

系统支持以下草图对象：

- Line
- Circle
- Rectangle

当前版本中草图主要用于：

- 表达二维建模意图；
- 作为 Extrude 的输入源。

#### 5.2.3 Extrude

系统支持将 Rectangle 和 Circle 草图转换为三维实体。实现方式为：

- 根据草图参数生成 `THREE.Shape`
- 调用 `THREE.ExtrudeGeometry`
- 将结果加入对象历史与三维场景中

#### 5.2.4 Cut

为了保证演示稳定，Cut 未采用完整实体布尔差，而是采用“切除区域可视化标记”的简化实现。这一实现方式已经在文档中明确标注为 MVP 工程取舍。

### 5.3 协同功能实现

#### 5.3.1 房间机制

系统以 `roomId` 为协同作用域。多个用户进入同一 `roomId` 后，共享同一份房间状态。

#### 5.3.2 操作广播

后端通过 WebSocket 广播以下操作：

- `add`
- `update`
- `delete`
- `replace-state`
- `cursor`

#### 5.3.3 冲突处理

当前采用“最后写入优先”策略。该策略虽然简单，但足以支撑课程设计场景下的双窗口协同演示，并显著降低实现复杂度。

### 5.4 保存、加载与版本管理

每次用户点击 Save 时：

- 当前房间状态会保存到：
  - `backend/data/projects/{roomId}.json`
- 同时生成一个版本快照：
  - `backend/data/projects/{roomId}/versions/{timestamp}.json`

系统同时支持：

- Load：恢复最近一次已保存状态
- Restore：恢复指定历史版本
- Undo / Redo：客户端局部历史回退与重做

### 5.5 智能辅助建模入口

为增强创新性，本系统增加了 Smart Assist 输入框。用户可以输入：

`创建 box width 4 height 2 depth 3`

系统通过轻量规则解析器识别对象类型与参数，并自动生成对应对象。该实现不依赖外部 AI 服务，但体现了“自然语言驱动 CAD 操作”的扩展方向。

## 6. 系统测试与结果分析

### 6.1 测试内容

测试内容包括：

- 前后端安装与启动
- 双窗口加入同一房间
- Box / Cylinder 新建同步
- 参数修改同步
- Save / Load
- Undo / Redo
- 版本恢复

### 6.2 测试结果

测试表明：

- 前后端可在本地正常运行；
- 房间内对象新增与参数修改可实时同步；
- Save 后可以生成 JSON 主文件与版本快照；
- Load 与 Restore 能恢复已保存状态；
- Undo / Redo 可以完成本地历史控制；
- Smart Assist 可以完成简单对象命令创建。

### 6.3 结果分析

系统在课程设计要求范围内已经实现了较完整的功能闭环，具有以下优点：

- 演示稳定
- 结构清晰
- 便于讲解
- 可扩展性较强

不足之处包括：

- 未集成工业级几何内核
- Cut 为简化实现
- 协同冲突策略较简单

## 7. 创新点与工程取舍

本项目的创新点主要体现在：

1. 在较低复杂度下实现了云协同 CAD 的完整演示闭环；
2. 采用前端重计算、后端轻协同的课程设计友好架构；
3. 增加了自然语言辅助建模入口；
4. 在架构中预留了 OpenCASCADE/Wasm 和 AI 扩展位。

工程取舍主要体现在：

- 放弃工业级布尔运算复杂度，优先保证 MVP 稳定交付；
- 放弃数据库部署复杂度，优先使用 JSON 实现版本化；
- 放弃外部 AI 服务依赖，优先实现本地可控的智能入口。

## 8. 总结与展望

本文设计并实现了一个基于云协同思想的机械 CAD 系统 MVP。系统完成了基础建模、草图表达、拉伸生成、实时协同、保存加载、版本管理和轻量智能辅助建模等功能，达到了课程设计“可运行、可演示、可说明”的目标。

后续可从以下方向进一步扩展：

- 接入 OpenCASCADE/Wasm 提升几何计算能力；
- 引入 PostgreSQL 与 OBS 提升存储能力；
- 增加 JWT 登录认证；
- 引入 LLM 完善自然语言建模与参数推荐。

综上，本系统在课程设计范围内实现了较好的工程完整性和答辩可展示性，具有进一步拓展为更完整协同 CAD 平台的基础。
