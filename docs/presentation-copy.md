## 第 2 页：课题背景

正文可直接放：

传统机械 CAD 系统多以单机软件为主，虽然功能强大，但在多人协同编辑、轻量部署和跨平台访问方面存在不足。随着 Web 图形渲染、云协同和实时通信技术的发展，基于浏览器的协同 CAD 系统成为具有研究与工程价值的方向。

本课设的目标是在本地环境下实现一个可稳定演示、结构清晰、便于扩展的云协同机械 CAD 系统 MVP。

## 第 3 页：需求分析

正文可直接放：

本系统需要完成以下核心需求：

- 提供基本 CAD 操作界面，包括工具栏、对象树、三维视图和属性面板；
- 支持 Box、Cylinder、Sphere 等基础建模；
- 支持 Rectangle、Circle、Line 等草图对象；
- 支持 Extrude、简化 Cut、Save/Load、版本管理；
- 支持基于房间的多人实时协同编辑。

## 第 4 页：总体架构

正文可直接放：

系统采用“前端重计算、后端轻协同”的总体架构。前端负责建模状态、三维渲染和交互逻辑，后端负责房间状态维护、WebSocket 消息广播以及项目数据的持久化存储。

当前使用 Three.js 完成几何体渲染，并通过本地 JSON 文件保存项目和版本快照。后续可扩展到 OpenCASCADE/Wasm、PostgreSQL 和云端对象存储。

## 第 5 页：前端模块设计

正文可直接放：

前端主要由以下模块组成：

- Toolbar：负责建模与项目操作命令；
- Feature Tree：显示建模历史与对象列表；
- CanvasViewport：负责 Three.js 场景渲染、对象拾取与光标显示；
- InspectorPanel：负责属性编辑与版本恢复；
- Zustand Store：统一管理对象状态、协同消息和 Undo/Redo。

## 第 6 页：后端与协同机制

正文可直接放：

后端基于 Node.js + Express + ws 实现。系统以 `roomId` 作为协同作用域，前端通过 WebSocket 加入房间并发送建模操作。后端负责更新房间状态，并广播 `add/update/delete/cursor/replace-state` 等操作消息给其他客户端。

在冲突处理上，当前采用“最后写入优先”策略，以保证课程设计场景下的实现简单性与演示稳定性。

## 第 7 页：核心功能实现

正文可直接放：

系统已实现以下核心功能：

- Box、Cylinder、Sphere 基础三维建模；
- Line、Circle、Rectangle 草图对象；
- 基于 Rectangle / Circle 的 Extrude；
- 简化 Cut 效果；
- Save / Load / Version Restore；
- Undo / Redo；
- 双窗口实时协同与在线用户显示。

## 第 9 页：创新点

正文可直接放：

本项目的创新点主要体现在以下三个方面：

1. 在较低复杂度下实现了云协同 CAD 的可运行 MVP；

3. 增加了 Smart Assist 自然语言辅助建模入口，并为 AI 辅助设计和 OpenCASCADE/Wasm 几何内核预留了扩展接口。

## 第 11 页：不足与后续优化

正文可直接放：

当前系统仍存在以下不足：

- 未接入工业级几何内核；
- Cut 为演示级简化实现；
- 协同冲突控制策略较简单；
- 未实现正式用户鉴权。

后续可从以下方向优化：

- 接入 OpenCASCADE/Wasm；
- 引入 PostgreSQL 与 OBS；
- 增加 JWT 登录认证；
- 引入大模型实现更强的自然语言建模与参数推荐。
