# 答辩演示稿与高频问答

## 1. 3~5 分钟演示稿

### 开场（30 秒）

大家好，我的课程设计题目是《基于云的协同机械 CAD 系统设计与实现》。本项目的目标不是实现完整工业级 CAD，而是完成一个可本地稳定运行、支持多人协同演示、并且便于课程答辩说明的 MVP 系统。

### 第一步：说明系统定位（20 秒）

本系统采用 React + Three.js 构建前端 CAD 界面，Node.js + Express + WebSocket 构建后端协同服务，使用本地 JSON 进行项目保存与版本快照。整体采用“前端重计算、后端轻协同”的架构。

### 第二步：演示登录与进入同一房间（20 秒）

现在我打开两个浏览器窗口，分别输入两个不同用户名，并进入同一个 `roomId`。这一步说明系统支持基于房间的协同编辑。

### 第三步：演示基础建模（40 秒）

在第一个窗口中，我依次创建：

- Box
- Cylinder
- Rectangle 草图

左侧 `Feature Tree` 会记录建模历史，右侧 `Properties` 可以编辑当前对象参数。中间是 Three.js 3D 视图区。

### 第四步：演示实时协同（40 秒）

我在第一个窗口修改 Box 的宽度或位置，第二个窗口会实时同步更新。说明后端通过 WebSocket 广播操作增量，房间内其他客户端会重放操作并刷新视图。

### 第五步：演示草图到实体（30 秒）

我选中 `Rectangle` 或 `Circle` 草图后点击 `Extrude`，系统会生成对应 3D 实体。这样完成了从二维草图到三维模型的典型建模流程。

### 第六步：演示智能辅助入口（30 秒）

在 `Smart Assist` 输入框中输入类似：

`创建 box width 4 height 2 depth 3`

系统会自动解析命令并创建对象。这是本项目在创新性上的一个轻量扩展点，体现了后续接入 AI 辅助设计的可能性。

### 第七步：演示保存、加载与版本恢复（40 秒）

点击 `Save` 后，后端会将当前房间模型保存到本地 JSON 文件，并生成一个时间戳版本快照。右侧可以看到版本列表。之后我可以执行 `Load` 或 `Restore`，恢复之前的保存状态。

### 第八步：演示 Undo / Redo（20 秒）

我删除一个对象，再使用 `Undo / Redo`，可以看到对象状态按客户端历史回退与重做。

### 结束总结（20 秒）

这个系统完成了课程设计要求中的核心能力：基础建模、实时协同、保存加载、版本管理，以及可扩展的架构预留。虽然当前没有集成工业级几何内核，但已经满足课程设计的可演示、可运行、可说明目标。

## 2. 老师高频问题与回答

### 2.1 为什么没有直接使用 OpenCASCADE/Wasm？

答：

课程设计更强调“完整系统设计与可演示交付”。OpenCASCADE/Wasm 集成复杂、调试成本高，而本项目优先保证本地稳定运行、协同可演示、报告好写。因此当前版本使用 Three.js 几何体完成 MVP，同时在架构中明确预留了 OpenCASCADE/Wasm 几何内核替换位。

### 2.2 为什么采用“前端重计算、后端轻逻辑”？

答：

因为本项目的重点是协同 CAD 系统，而不是服务端几何计算平台。前端负责建模和渲染，后端只维护房间状态、广播操作和保存版本，这样实现难度更低，演示链路更清晰，也便于后续替换几何内核。

### 2.3 为什么使用本地 JSON，而不是数据库？

答：

课程设计环境是本地 macOS，无 Docker、不做远程部署，因此使用 JSON 存储可以显著降低部署复杂度，并且更方便展示 `Save / Load / Version Snapshot` 概念。后续可平滑迁移到 PostgreSQL 和 OBS。

### 2.4 你们的协同冲突怎么处理？

答：

当前采用“最后写入优先”策略。也就是后端按最新收到的操作覆盖对象状态。这个策略实现简单，适合课程设计 MVP，重点是保证演示稳定。后续若扩展到工业级协同，可以引入 OT 或 CRDT。

### 2.5 版本管理是怎么实现的？

答：

每次点击 `Save` 时，后端会做两件事：

- 更新 `backend/data/projects/{roomId}.json`
- 在 `versions/` 目录下生成一个带时间戳的快照文件

这样既有当前状态，也能查看历史版本并恢复。

### 2.6 你们的 Cut 为什么不是完整布尔差？

答：

完整布尔差需要更强的几何内核支持。为了保证课设周期内的稳定交付，当前使用“可视化切除标记”作为简化实现，并在文档中明确标注为 MVP 方案。这种处理是工程取舍，不是遗漏。

### 2.7 Smart Assist 算不算 AI？

答：

当前实现是“AI-inspired”的轻量智能辅助入口，采用规则解析完成自然语言近似命令到建模操作的转换。它的意义在于验证了“自然语言驱动 CAD 命令”的交互方向，并为后续接入真正的大模型保留了接口语义。

### 2.8 这个项目的创新点是什么？

答：

创新点主要有三点：

- 将云协同 CAD 的关键闭环在本地轻量实现出来
- 采用前端重计算、后端轻协同的架构取舍
- 增加自然语言辅助建模入口，并在架构中预留 OpenCASCADE/Wasm 与 AI 扩展方向

## 3. 现场演示顺序速记卡

1. 登录两个窗口，同房间
2. 新建 Box / Cylinder
3. 修改 Box 宽度，展示同步
4. 新建 Rectangle，展示草图
5. 执行 Extrude
6. Smart Assist 输入命令并运行
7. Save，展示版本列表
8. Delete + Undo / Redo
9. Load 或 Restore
10. 打开架构图，结束总结
